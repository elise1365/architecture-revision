Process

The unit of work in a computer system/ a sequence of states resulting from the action of a set of instructions on the states as they develop

Two aspects of any process:

  1. Static part - resources allocated to the process
  2. Dynamic part - a program in action

Must be possible to represent one task (static) and a variable number of threads (dynamic)

How does the OS manage processes

  • The OS maintains a data structure known as PCB (process control block) for each process. The PCB contains useful information such as the current process state, the next instruction to perform and currently allocated devices to the process
  • PCB allows OS to manage lots of different processes

Multithreading

Need for this is caused by OS everhead

A number of paths of execution through the program at the same time. If one thread is blocked, another can execute

No need to save and restore full state of the machine, as its using tje sa,e memory, just jumping to another location in the program code

Each thread must maintain some of its own state information

Multi threading enables the processing of multiple threads at one time, rather than multiple processes

Types of threads

  • User threads - above the kernel, without kernel support. The threads that application programmers use in their programs
  • Kernel threads - are supported within the kernel of the OS itself. OS handles all the switching between threads, all modern OSs support kernel level threads, allowing the kernel to perform multiple simultaneous tasks/to service multiple kernel system calls simultaneously

Threads

Thread is in one of 2 different states

  1. Run - ready and able to do some work
  2. Wait - unable to use the CPU, waiting for some event

A thread may stop using the CPU for 3 reasons:

  1. Voluntarily - wait state
  2. Termination - terminating a thread/whole process
  3. Preemption - when a thread has used up its share of time/ CPU is needed to handle urgent work, the OS takes the processor from a thread

Context switching

Reallocating a processor from one thread to another

Called when:

  • A thread loses control of the processor and moves into the wait state
  • A timer interupts to tell a thread its used up its time slice

It is essential to save the current state of the machine so when the thread is eligible to run again, it can be set up correctly and the thread will continue

Thread scheduling

Main objective of the scheduler is to see that the CPU is shared among all contending threads as fairly as possible

Scheduler tries to make the responses time as short as possible

Thread priority - order all ready threads by priority

Thread multilevel priority queues - split queues into high and low priority queues
