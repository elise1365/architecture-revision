First come first served scheduling

The process that requests the CPU first is allocated the CPU

FIFO queue

Evaluation

  • + easy to use/ implement
  • - poor in performance
  • - high wait time

Shortest job first scheduling

The process with the shortest execution time goes first

Preemptive or non preemptive

Priority first scheduling

Priority number associated with each process, SPU allocated to process with highest priority (lowest number)

Preemptive or non preemptive

PRoblem with priority scheduling is starvation/indefinite blocking

Aging solves this problem by gradually increasing the priority of processes that wait in the system for a long time

Round robin scheduling

Designed specifically for time sharing systems

Similar to FCFS but preemption is added, each process gets a small unit of CPU time (time slice) and after this time the process is preempted and either terminates or is added to the end of the FIFO queue

Multilevel queue scheduling

Ready queue split into separate queues, foreground (interactive) and background (batch) processes

Processes added permanently to one queue

Each queue has its own scheduling algorithm

Multilevel feedback queue scheduling

Process is not permanently assigned to one queue
