CPU scheduling

  • By switching the CPU among processes, the OS can make the computer more productive
  • A process is executed until it must wait, usually for a I/O request
  • Several processes are kept in memory st one time. When a process has to wait, the OS takes the CPU away from it and gives the CPU to another process

I/O cycle

  • Process execution consists of a cycle of CPU execution + I/O wait
  • Process execution begins with a CPU burst, then I/O burst etc
  • The final CPU burst ends with a system request to terminate execution

CPU scheduler

When the CPU becomes idle, the OS must select one of the processes in the queue ready to be executed. The selection process is carried out by the CPU scheduler

Non preemptive scheduling

Process cannot be stopped by other processes, new processes only selected when a process terminates or a process switches from running to waiting

Preemptive scheduling

When a process with higher priority arrives, running of current process may be stopped. Special hardware is required for preemptive scheduling

Dispatcher

Takes a process to the desired queue

Gives a process control after it has been selected by the short term cheduler. Involves switching context, switching to user mode or jumping to the proper location to restart a program

Scheduling criteria

  • CPU utilisation - keep the CPU as busy as possible
  • Throughput - number of processes that complete their execution per time unit
  • Turnaround time - amount of time to execute a process, exit time - arrival time
  • Waiting time - amount of time a process has been waiting in the ready queue, turnaround time - burst time
  • Response time - amount of time it takes from when a request was submitted to when the response is produced
