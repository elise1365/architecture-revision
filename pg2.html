Expanded IAS structure

  • MBR = contains a word to be stored in memory or sent to I/O unit or is used to recieve a word from memory/ I/O unit
  • MAR = specifies the address in memory of the word to be written from or read into the MBR
  • IR = contains the 8 bit opcode being executed
  • IBR = temporarily holds the right hand instruction from a word in memory
  • PC = contains the address of the next instruction to be fetched
  • AC + MQ = temporarily holds operands and results of ALU operations

Three main components of IAS:

  1. Single read write memory
  2. Contents of this memory are addressable by location
  3. Execution occurs in sequential fashion (unless explicitily modified) from one instruction to the next

Fetch execute cycle

Fetch = opcode of the next instruction is loaded into IR, address portion loaded into MAR, this instruction may be taken from the IBR or memory (by loading a word into the MBR then down to the IBR, IR and MAR)

Execute = control cicuitry interprets the opcode and executes the instruction by sending out the appropriate control signals to cause data to be moved or an operations to be performed by ALU
