The history of operating systems

1940's - no OS, the programmer was also the user. Programs and data entered in binary using switches at front of machine. Output by lights. Programmer was the OS

1950's - specialist perators who tended to the machine, fed the programs and delivered output to the programers. Operators were the interface between programmer and hardware

1960's - programmer punched instructions on control cards which were inserted at the apropriate places before and after program cards and data cards

Late 1960's + 1970's - first attempts to provide interactive use of a compuer. The issue of protecting one program from another became important

1980's - networking and communication

1990's - move to open systems, a move to build a generic OS which will run on any hardware

OS

An OS is a program that is run every time a computer is turned on. It is siftware which makes the hardware more useful & user friendly

OS is below the application program and above the hardware, it acts a middleman between these two

OS jobs:

  • Helps a user to develop and run programs by providing a convenient environment
  • Starting and stopping programs and sharing CPU between them
  • Managing memory - identifies and arranges which parts of memory are in use and which parts are free
  • I/O management - covers up differences between alternative makes and models of devices
  • File systems/organisation
  • Protection - protects from program interference
  • Networking - covers up the difference between machines
  • Error handling and recovery

GUI and command interpreters

Applications that sit on top of the OS

System service interface

A set of functions provided by an OS to enable a user to request a service from it

Kernel

The central component of OS

Makes the software to interact with the hardwarw to get a specific task done

Decides the amount of resources to be used by every application + decides what programs to be executed and in what order

Has separate space in memory so it functions independantly

Acts as a central authority which guides memory and keeps an eye on all the hardware and software data flow

System call - every process which starts on a system demands the resources from the kernel

Monolithic kernel

The entire OS is run as a single large program in kernel space

User services and kernel services are both kept in the same address space

  • + Higher performance
  • - Higher risk of system crash

Microkernel

Based on a smaller kernel that provides only the essential OS services

User services and kernel services are kept in separate address spaces

  • + smaller in size
  • - greater access time
  • - slow execution time

Modes

2 modes:

  1. Kernel mode - the CPU can execute all of its instructions including extra priviledged instructions. If a program is executed in kernel mode, it has direct access to memory, hardware and such resources
  2. User mode - the CPU can only execute a subset of its instructions. If a program is executed in user mode, it does not have access to memory, hardware and such resources

Types of OS

  • Batch systems - data and commands to manipulate the program and data are all submitted together as a job. Little interaction between users and executing a program
  • Interactive systems - uses keyboard, mouse and screen. Makes it possible to intervene directly while a program is being developed or as it is running
  • General purpose OS - combination of boatch and interactive OS
  • Network OS - to share resources such as printers and databases across a network
  • Distributed systems - consists of a group of machines acting together as one
  • Specialist systems - dedicated to processing large volumes of data which are maintained in an organised way

Functions of an OS

  • Process management
  • Memory management
  • I/O management
  • File storage
  • Network management
