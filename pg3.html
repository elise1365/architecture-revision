Interconnection structures

The collection of paths that connect modules

The design of the interconnection structure depends on the exchanges that need to be made

Types of exchanges:

  • Memory - typically a memory module consists of N words of equal length, each word is assigned a unique numerical address, a word can be written or read into memory, the location for an operation is specified by an address
  • I/O module - 2 operations (read/write), may control more than one external device, we can refer to each of the interfaces to an external device as a port and give each a unique address. There are external data paths for the input and output of data with an external device. It may be able to send interupt signals to the processor
  • Processor - reads iinstructions and data, wirtes data after processing and uses control signals to control the overall operation of the system. Recieves interupt signals

The interconnection structre must support:

  • Memory to processor transfers - the processor reads an instruction/unit of data from memory
  • Processor to memory transfers - processor writes a unit of data to memory
  • I/O to processor transfers
  • Processor to I/O transfers
  • I/O to/from memory - I/O us allowed to exchange data directly with memory without going through the processor

Bus

A bus is a communication pathway connecting 2 or more devices

It has a shared transmission medium, so multiple devices can connect to the bus, and a signal transmitted by one device is available for reception from all other devices

Only one device can successfully transmit at a time

A bus consists of multiple communication pathways, each transmits a signal representing binary 1/0

Bus operation:

If one module wishes to send data to another it must obtain the use of the bus and transfer the data via the bus

If one module wishes to request data from another module, it must obtain the use of the bus, and transfer a rquest to other modules over the appropriate control and address lines

Functional groups

Lines are classed as one of these

  • Data lines = provide a path for moving data among system modules
  • Address lines = designate the source or destination of the data on the data bus
  • Control lines = control the access to and the use of the data and address lines, control signals

Point to point interconnection

Contemporary systems rely more on p2p rather than shared bus systems because there are electrical constraints that come from buses

  • + Lower latency
  • + higher data rate
  • + better scalability
